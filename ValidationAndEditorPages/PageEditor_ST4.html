Student 4, Profile & Sitemap

Profile Page

Technical Description

The Profile page was built using semantic HTML and styled with both global and custom CSS. The layout is divided into visually distinct cards for user information, skills, volunteering, interests, calendar, and contact form. Each card is styled with borders, padding, and hover effects to enhance interactivity.

A key feature is the use of CSS percentage bars to visually represent skill levels. These are implemented using nested elements with dynamic width values. Hover effects are applied to icons and cards using CSS transitions to improve user engagement.

Accessibility

  • Using alt text for the user avatar and icons.
  • Structuring the page with semantic HTML elements like section, header, and footer.
  • Ensuring all form fields in the "Contact Me" section have associated label elements for screen reader compatibility.

Sitemap Page

Technical Description

The Sitemap page was implemented using inline SVG to visually represent the websiteâ€™s structure. Each page is shown as a rectangle node, and lines (line) are used to connect parent and child pages. Nodes are interactive, with hover effects and clickable links using anchor elements with xlink:href attributes.

CSS was used to style the nodes with colors, rounded corners, and transitions. The SVG is made responsive using the viewBox attribute, allowing it to scale across devices.

Accessibility

  • Title tags inside each SVG anchor element to describe the node.

Content Page

Technical Description

The Content page is structured using semantic HTML with clear headings (h1 to h4) and internal links for navigation. A "Go to Top" button is implemented using a fixed-position anchor link. The page includes multiple sections with relevant content and styled images using CSS for borders and responsiveness.

Accessibility

The page uses a logical heading structure for screen readers. All images include meaningful alt text. All links and buttons are keyboard accessible. Semantic HTML elements are used to ensure compatibility with assistive technologies, and the page maintains good color contrast and focus indicators.

Challenges and Lessons Learned

One of the main challenges was creating a responsive SVG sitemap that worked well across screen sizes. It required careful use of the viewBox attribute and CSS styling. Another challenge was implementing interactive elements like hover effects and progress bars in the Profile page while maintaining accessibility.

Through these tasks, I learned how to:

  • Use SVG effectively for visual diagrams
  • Apply accessibility best practices
  • Integrate multiple components into a consistent website template

Compliance

All pages were developed in compliance with Janet regulations:

  • Content is original or properly referenced
  • Images and icons are sourced from royalty-free platforms like iStock
  • Accessibility standards (WCAG 2.1) were followed using semantic HTML, alt text, and keyboard navigation
